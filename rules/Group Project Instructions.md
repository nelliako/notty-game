# Group Project Instructions

### Completion requirements

#### Overview

In groups of 4-5 students, you will work together to produce a Python program for playing the card game _Notty_.

#### Description of the Game

A [description of the game can be found in this document](https://moodle.nottingham.ac.uk/mod/resource/view.php?id=8707885).

#### Task

The overall task in this coursework is to create a Python program to allow a human player to play the _Notty_ game against one or two computer opponents. You must use the Pygame framework to implement this. [Details of Pygame can be found at this link](https://www.pygame.org/), and it will be introduced in the lectures.

You should include the following in your solution:

*   A simulation of the collection of cards and the basic gameplay, including the initial dealing of the cards, the turn-taking structure, the playing of the four actions, checking that actions are valid and denying ones that are not, maintaining the deck and shuffling it when needed, checking for the winning condition, and declaring the winner.
*   You should use the facilities in Pygame to create a visual interface to the game, so that the user can see the cards by all players and play the game by clicking on cards/buttons or similar. The various actions by the human and computer players should be visualised appropriately in the interface.
*   You should implement computer players for the game. Initially, this can consist of the computer player working out which of the four actions are possible and choosing one of the possible actions at random. Later on, you could add sophistication to the computer players: for example, the computer might calculate which is the best of the four actions to take (or whether it is better to pass); how many cards to draw when taking a draw action; which player to take from when taking a random card from another player; and, which is the best valid group to discard when there is a choice of possible valid sets. This is an open-ended part of the assessment, and you should use your creativity to come up with interesting strategies for the computer players to use. For the highest marks, you should implement multiple computer players, each taking a different approach, and the user can choose which players to play against.
*   You should add a button to the interface called _Play for Me_ which makes the computer player take your turn for you.
*   When a player (human or computer) wins, the program should display a congratulations message.

All of the lab sessions from 11th November until 3rd December will be set aside for work on the group projects (apart from when you are doing your individual coursework interview). You will not be able to complete the task just in the lab time: you will need to work outside of that time too. It is expected that each team member will spend around 10-15 hours each week on this project between 11th November and 5th December.

#### Marking Criteria

The following criteria will be used in assessing the coursework:

| Criterion | Weight |
| --- | --- |
| Correctness and completeness of the gameplay | 20% |
| Quality of the user-interface | 40% |
| Quality of the computer players | 20% |
| Quality of the demonstration | 20% |

Grades will be given according to the following guidelines:

| 80 and above | The submitted work is designed and implemented to a professional standard, and it goes beyond regular expectations in quality of design and implementation, particularly in areas where you have some freedom such as the computer players and visual interface. |
| --- | --- |
| 70-79 | All aspects of the task have been completed to a very high standard. |
| 60-69 | Almost all aspects of the task have been completed to a good standard, perhaps with a few minor problems. |
| 50-59 | Most aspects of the task have been completed to a competent standard, perhaps with a few problems. |
| 40-49 | Some attempt has been made at the task, but important aspects of the functionality are missing and/or there are significant problems. |
| 20-39 | A small attempt has been made at a few aspects of the task, but the majority is missing, wrongly designed/implemented, or erroneous. |
| 0-20 | Minimal or no attempt at the task, or no submission. |

A grade will be given to the group, which will be adjusted based assessment of each students’ contribution to give a final mark. If you did not participate in the project, then you will receive a mark of zero.

#### Submission and Demonstration

By 3pm on 5th December, each group should submit their code on Moodle. A submission folder will be created nearer the time.

Due to the group nature of this project and the assessment by group presentation, you will not be able to make a late submission. Therefore, if you have Extenuating Circumstances meaning that you miss the majority of the project work time, please ask for the assessment to be set aside on your Extenuating Circumstances form rather than asking for an extension. If you have a requirement for adaptation because of a support plan, please contact us early in the project and we will make appropriate adaptations.

In the week 8th\-12th December, each group will book a slot to demonstrate their program, explain how it works, and discuss how their group went about the process of designing and implementing their solution. Where possible, these will be done during the usual lab times.

Please contact the module convenors, [Colin.Johnson@nottingham.ac.uk](mailto:Colin.Johnson@nottingham.ac.uk) and [Daniel.Karapetyan@nottingham.ac.uk](mailto:Daniel.Karapetyan@nottingham.ac.uk) if there are any concerns about bullying, abuse, discrimination or harassment within the group.

#### Academic Integrity

This is a group assessment that should consist of the unaided work by the group. If your solution makes use of someone else’s code (e.g. our code from the classes, examples taken from forums, open-source projects, etc.), please make it clear which aspects of the code are not your group’s work through the use of comments that identify where the code is from and how it has been adapted for your submission. The University has [detailed advice about academic integrity](https://www.nottingham.ac.uk/studyingeffectively/referencing/integrity/index.aspx), and submissions that demonstrate a lack of that integrity will be treated under appropriate disciplinary procedures

#### Use of AI

Your code should be written without the use of AI coding assistants, and you should be able to explain all of the code that you have written. We may test your understanding of the details of the code during your group demonstration.
